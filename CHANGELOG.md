# 更新日志

[English](CHANGELOG.en.md) | 简体中文

Noteva 的所有重要变更都会记录在这里。

## [v0.3.0] - 2026-04-28

### 后端质量复查
- **后端 API 二次审查修复** - 围绕管理端和公开端 API 继续收紧分页、参数校验、错误响应、资源边界和状态刷新，降低重构后潜在的异常输入、旧状态和不一致响应风险。
- **插件、主题与更新链路稳定性打磨** - 继续整理 GitHub 安装/更新、商店状态、在线更新确认和本地缓存刷新，让版本检测和扩展包状态更贴近实际安装结果。
- **数据访问与运行时边界补强** - 针对文章、分类、标签、评论、文件、设置、插件数据和 WASM worker 等路径补齐边界处理与失败反馈，减少隐藏失败和脏状态。
- **登录 Cookie 与 2FA 安全边界加固** - 登录、注册和 2FA 验证统一使用站点 HTTPS 配置与可信代理判断 `Secure` Cookie，不再直接信任可伪造的转发协议头；2FA challenge 增加失败次数限制，并阻止已启用账号重复覆盖密钥。
- **文章、筛选和评论输入校验收紧** - 文章 slug 在创建和更新时统一规范化并拒绝路径型危险值；分类/标签筛选遇到不存在的 slug 会返回空分页结果；评论内容在插件过滤前后都会校验空值和长度上限，避免 hook 写入异常内容。
- **插件操作与 ZIP 导入路径保护补齐** - 插件卸载和更新前校验插件 ID 目录名，同时保留中文/Unicode ID 兼容；备份恢复和 Markdown ZIP 导入增加条目数、单文件大小、总解压体积、符号链接和路径穿越检查。
- **Windows 在线更新替换流程修复** - Windows 平台不再尝试在进程运行中直接替换当前可执行文件，改为写入 `.new.exe` 与更新辅助脚本，在旧进程退出后完成替换并重启，降低更新失败和旧版本残留风险。

### 管理后台
- **管理页面体验复查修复** - 针对文章、评论、文件、导航、设置、安全日志、插件和主题等页面修复审查中发现的交互细节、加载状态、批量操作反馈和错误提示问题。
- **文章列表公开链接修复** - `/manage/articles` 的预览和复制链接现在只对已发布文章启用，并按站点 `site_url` 与 `permalink_structure` 生成真实公开地址，避免草稿/归档文章复制出不可访问链接或在 ID 永久链接模式下跳错地址。
- **文章列表批量操作补全** - 文章列表多选后会显示批量工具条，支持批量发布、转为草稿、归档、删除和取消选择；批量删除增加二次确认与部分失败反馈，让全选/多选不再只是无效状态。
- **分类/标签同页布局对齐** - 统一 `/manage/taxonomy` 中分类与标签两侧的信息密度、卡片结构、列表高度和操作区域，避免合并后视觉不对称。
- **刷新按钮反馈统一** - 安全日志和自定义页面刷新按钮对齐插件页样式，请求中仅按钮图标旋转，成功后短暂显示绿色完成状态；手动刷新不再触发顶部同步提示、进度条和表格透明度闪烁。
- **评论列表刷新闪烁收敛** - 评论管理页手动刷新改为保留当前列表并静默更新数据，不再因为快速刷新挂载同步条、全表透明度和额外 loading 状态。
- **自定义页面管理体验补齐** - 自定义页面列表接入统一后台页头、卡片容器、首次加载状态、后续同步状态和乐观删除反馈，编辑、创建、删除后的列表状态更稳定。
- **安全日志筛选与刷新链路整理** - 安全日志筛选条件改为显式应用，刷新请求复用当前分页和筛选参数，并按当前后台语言格式化登录时间。
- **文章编辑器表情选择器优化** - 修复表情弹窗使用隐藏元素定位导致的漂移问题；桌面端按真实按钮定位并限制在视口内，移动端改为底部弹层并避免打开时强制拉起键盘。
- **Markdown 编辑器稳定性继续打磨** - 优化预览请求、媒体库搜索、上传入口和编辑/预览区域滚动表现，减少快速输入、切换和上传时的闪烁与旧请求覆盖。
- **图片网格编辑命令** - Markdown 编辑器工具栏新增图片网格按钮，支持将选中内容包裹为 `[grid]...[/grid]`，未选中内容时插入空网格块，由作者自行决定是否启用多图网格排版。
- **文章缩略图清空修复** - 编辑文章时点击缩略图 `X` 后，更新接口现在能区分未传字段、传 `null` 清空和传字符串设置，刷新后不再恢复旧缩略图。

### 默认主题
- **默认主题体验复查修复** - 继续收敛首页、归档、分类、标签、文章页、评论区、页头页脚和主题交互中的状态同步、SDK 等待和边界显示问题。
- **默认主题内置语言扩充** - 默认主题新增日语、韩语、法语、德语、西班牙语、巴西葡语、俄语和意大利语语言包，并统一语言列表、浏览器语言匹配和 `html lang` 同步。
- **分类、标签和归档页面一致性打磨** - 统一内容索引页面的数据拉取、空态、分页和卡片密度，让默认主题在多内容量场景下更稳定。
- **`[grid]` 图片网格渲染支持** - 后端 Markdown 渲染支持独立 `[grid]...[/grid]` 图片块，默认主题和后台预览补齐响应式网格样式，多图在移动端默认两列、桌面端默认三列，四图保持 2x2 展示。

### 前端质量复查
- **内置语言包扩充** - 管理后台新增日语、韩语、法语、德语、西班牙语、巴西葡语、俄语和意大利语内置语言包，常用语言不再依赖用户手动上传自定义语言包。
- **语言检测与回退策略统一** - 后台和默认主题都会优先匹配浏览器语言，支持精确匹配和语言前缀匹配；未命中时回退英文，已选择语言会同步到 `html lang` 并持久化。
- **多语言加载性能优化** - 管理后台保留中文、繁中、英文为首屏内置消息，其余语言按需懒加载，减少额外语言包对初始后台体积的影响。
- **自定义语言链路修复** - 管理后台语言切换器改为读取运行时语言列表，自定义语言包加载后可直接显示；当前语言会同步到 `html lang`，插件和主题设置面板的多语言字段会按应用内当前语言渲染。
- **文章编辑与列表请求稳定性优化** - 新建文章草稿恢复会正确同步到懒加载 Markdown 编辑器，初始化数据加载增加卸载保护；默认主题首页和文章列表 hook 增加请求顺序保护，避免旧请求覆盖新搜索或分页结果。
- **上传与复制反馈修复** - FormData 请求统一让浏览器生成 multipart boundary，不再传递 `Content-Type: undefined`；文件管理复制链接改为真实成功/失败反馈。
- **前端 HTML 渲染兜底清洗** - 文章、自定义页面、页脚、后台 Markdown 预览和搜索高亮在前端增加轻量 HTML 清洗，过滤脚本标签、事件属性和危险 URL，作为后端清洗之外的防漏层。
- **前端国际化与硬编码文案收敛** - 补齐语言切换、目录、图片上传、空状态、加载状态等 UI 文案的多语言翻译，减少后台和默认主题中的硬编码文案。

### 构建与版本
- **版本号统一为 0.3.0** - 同步更新 Rust crate、Cargo.lock、前端包、默认主题、SDK 内置版本和开发元信息中的项目版本。

## [v0.2.9] - 2026-04-27

### 插件与主题商店
- **商店职责收敛为列表源** - 插件和主题商店侧重提供名称、描述、仓库地址等基础索引信息，安装与更新回到 Noteva 自身的 GitHub 安装和检测链路。
- **插件/主题在线安装入口重构** - 后台“上传插件/上传主题”调整为“安装插件/安装主题”，弹窗内同时支持 ZIP 上传和 GitHub 仓库地址安装。
- **已安装项更新检测改进** - 插件和主题更新不再依赖商店更新接口，而是基于本地已安装清单中的仓库信息检测 GitHub 版本，并在可更新时显示更新操作。
- **中文/Unicode 插件 ID 兼容** - 前后端安装、更新和路径参数处理补齐编码与校验，避免中文插件 ID 在请求路径中被错误解析。

### 管理后台
- **分类与标签合并管理** - 侧边栏将“分类”和“标签”合并为一个入口，新增 `/manage/taxonomy` 同页管理分类与标签，旧 `/manage/categories` 和 `/manage/tags` 自动跳转到新页面。
- **标签管理界面收敛** - 移除后台标签云重复视图，保留更适合管理场景的标签列表、搜索、文章计数、单项删除和批量删除。
- **插件/主题刷新体验优化** - 插件和主题页面刷新时保留已有内容，去掉明显闪烁的整页进度条，减少重复加载造成的视觉跳动。
- **在线更新状态刷新修复** - 程序在线更新后清理旧版本检查缓存；管理端进入后台时会自动重新检查程序更新，不再复用持久化的旧检查结果，避免紧急版本提示被缓存延迟。
- **安装与更新交互整理** - 插件和主题的安装、检查更新、刷新等动作统一走更稳定的按钮状态和弹窗流程，降低误触和重复刷新感。
- **文章列表搜索改为后端检索** - `/manage/articles` 搜索词会传入后端 `keyword` 查询，不再只过滤当前页结果；筛选和搜索切换时自动回到第一页。
- **分类层级管理补齐** - 后台分类创建和编辑弹窗支持选择父分类，列表中展示父分类信息，并避免选择自身或子级作为父级。
- **文章编辑非法 ID 处理修复** - 进入无效文章 ID 时不再停留在无限 loading 状态，会提示加载失败并返回文章列表。
- **插件/主题商店状态刷新修复** - 上传、仓库安装、更新和删除插件/主题后会刷新商店缓存，避免已安装状态和更新按钮显示滞后。
- **编辑器媒体库搜索稳定性优化** - Markdown 编辑器媒体库搜索增加 debounce 与请求顺序保护，避免快速输入时旧请求覆盖新结果。
- **资料保存与文件批量删除反馈修复** - 保存个人资料后同步更新登录用户状态；文件批量删除会区分全部成功、部分失败和全部失败，并显示失败数量与部分文件名。

### 默认主题
- **文章页分类展示调整** - 默认主题文章详情页移除标题上方的分类展示，减少标题区域重复信息。
- **首页搜索改为全站后端搜索** - 默认主题首页搜索会通过 SDK `keyword` 查询后端文章接口，并支持搜索结果分页，不再只过滤当前页。
- **归档/分类/标签文章加载补全** - 默认主题归档页、分类详情页和标签详情页改为按页拉取全部匹配文章，移除固定 `pageSize: 100` 带来的截断风险。
- **分类层级展示补齐** - 默认主题分类页按父子层级展示分类，并在父分类卡片中提供子分类入口。
- **评论错误提示与导航外链加固** - 评论提交失败时展示服务端返回的具体错误；导航 external 链接渲染前校验协议，拒绝危险 URL。

### 后端 API 安全与稳定性
- **公开静态资源路径加固** - 修复主题静态文件、用户主题资源、上传文件、备份恢复和自定义语言包等路径处理中的穿越风险，统一使用安全相对路径、目录边界校验和 locale code 白名单。
- **2FA 登录链路修复** - 启用双因素认证的账号在密码验证通过后只返回短期 challenge，不再提前发放可用 session；验证码通过后才创建真实会话和 Cookie。
- **公开文章接口收敛为已发布内容** - 公开文章列表、搜索、分类文章、标签文章接口统一只返回已发布文章；后台文章列表改走受保护的管理接口，避免草稿和归档文章泄露。
- **公开写入接口权限收紧** - `/api/v1/cache/{key}` 公开路由仅保留读取，写入和删除移入登录保护路由；Markdown 渲染预览接口也改为登录后使用。
- **上传与下载内容安全增强** - 拒绝 HTML、JS、SVG、XML 等主动内容上传，并对已存在的主动内容上传文件强制下载、添加 `nosniff`，降低同源脚本执行风险。
- **GitHub 安装与更新加固** - 插件和主题 GitHub 安装仅允许 GitHub archive/release 地址，增加下载大小限制；更新时校验新包 ID/主题 short 必须与原目标一致，避免被替换成其他包。
- **在线更新完整性校验增强** - 自动更新下载增加包体和二进制大小限制，要求 `.sha256` 校验文件存在且格式合法，不再在缺少校验时继续更新。
- **公开设置脱敏修复** - 插件和主题公开 settings 只返回 settings schema 中声明且非 secret 的字段；缺少 schema 时不再暴露全部已保存设置。
- **请求来源识别收紧** - 登录日志、评论和点赞指纹改用真实连接来源 IP，不再信任客户端可伪造的 `X-Forwarded-For` / `X-Real-IP` 请求头。
- **主题 SDK 写请求 CSRF 补齐** - `noteva-sdk.js` 的 JSON 请求、上传请求和插件公开 API mutation 请求会自动携带 CSRF token，修复默认主题登录态写操作被 CSRF 中间件拒绝的问题。
- **Markdown HTML 输出清洗** - Markdown 渲染统一清洗危险标签、事件属性、危险 URL 和高风险 style，降低文章、独立页面和后台预览的 XSS 风险。

### 后端 API 行为修复
- **管理端分页参数限制** - 后台评论和登录日志接口限制 `page` 与 `per_page` 范围，避免负数、零或超大分页参数造成异常查询。
- **CORS 与 CSRF 边界修复** - CORS 在启用 credentials 时不再回退到 `*`；登出接口恢复 CSRF 校验，减少跨站写操作风险。
- **分类层级语义补齐** - 分类创建和更新支持 `parent_id`，分类错误按重复、未找到、校验失败等场景返回更准确的 API 错误；分类文章接口包含子分类文章。
- **文章写入校验修复** - 创建和更新文章时非法 status 会返回校验错误；未指定分类时查询默认分类，不再硬编码分类 ID。
- **reload 与文件删除响应修复** - 主题 reload 返回 `theme_count`，插件 reload 返回 `plugin_count`；文件删除遇到路径型 filename 会直接拒绝，不再静默取 basename。
- **在线更新后版本确认优化** - 管理端在线更新后会等待新进程返回目标版本再刷新页面，避免仍连接旧进程时显示旧版本和重复更新提示。
- **管理端文章创建路由补齐** - 后台新建文章改走 `/api/v1/admin/articles`，后端补齐对应 admin POST 路由，避免继续依赖旧的受保护 `/articles` 写入口。
- **导航目标校验补齐** - 导航服务端校验 builtin/page/external 类型目标，external 仅允许 `http://`、`https://`、`mailto:` 和 `tel:` 协议。

### 构建与版本
- **版本号统一为 0.2.9** - 同步更新 Rust crate、前端包、默认主题、SDK 内置版本和开发元信息中的项目版本。

## [v0.2.8] - 2026-04-27

### 插件系统
- **插件数据库 API 完善** - 为 WASM 插件补齐隔离数据库访问能力，支持插件在声明权限后执行受控 SQL 操作，并通过插件 ID 自动隔离数据边界。
- **插件数据库安全约束增强** - 收紧插件数据库调用的权限校验、语句校验和执行边界，避免插件越权访问核心数据或其他插件数据。

### 管理后台
- **侧边栏版本入口优化** - 在后台侧边栏增加当前版本与更新检查入口，保留更新日志查看能力，并移除设置页中重复的系统更新入口。
- **仪表盘最近文章收敛** - `/manage` 仪表盘最近文章固定展示最新 5 篇，加载态和渲染态保持一致。
- **文章编辑器滚动体验优化** - 新建和编辑文章时 Markdown 编辑区与预览区保持稳定高度，长内容改为内部滚动，避免编辑页面被内容持续撑高。
- **文章状态筛选修复** - `/manage/articles` 的已发布、草稿、已归档筛选改为后端按状态查询和计数，草稿与归档不再错误显示全部文章。
- **定时发布语义完善** - 文章设置定时发布时间后会作为草稿等待后台任务自动发布，发布时间可清空，实际发布后会自动移除定时标记。

### Bug 修复
- **文章 Emoji 破图修复** - Markdown 渲染不再把原生 Unicode Emoji 强制转换为外链 Twemoji 图片，修复星星等表情在默认主题文章表格中显示为破图的问题。

## [v0.2.5] - 2026-04-25

### 安全加固
- **禁用通用插件代理** - `/api/v1/plugins/proxy` 不再转发任意 URL，避免前端插件借公共代理触发 SSRF 或泄露服务端密钥。需要外部 API 的插件应通过插件设置保存用户配置，并由 WASM 后端在 `network` 权限下发起请求。
- **WASM Worker 沙箱收紧** - 为插件子进程补充内存、指令、请求/响应大小、日志/存储/数据库操作数量限制；HTTP 仅允许 `http/https`，禁止本地、内网、元数据地址和跳转。
- **插件/主题安装包校验增强** - ZIP/TAR 解包统一校验路径，拒绝路径穿越、符号链接、特殊文件、异常包名、超大条目和超大解包体积。

### 稳定性
- **核心迁移事务化** - 数据库迁移的 SQL 执行和 `_migrations` 记录写入放在同一事务中，并校验已应用迁移的版本、名称和连续性，避免半成功状态继续启动。
- **插件迁移事务化** - 插件 `migrations/` SQL 与 `plugin_migrations` 记录写入改为同一事务；失败时不再留下半创建表或错误记录。
- **安装覆盖顺序收紧** - 插件安装先定位并校验 `plugin.json` 和真实插件 ID，再覆盖目标目录，减少坏包误删已有插件的风险。

### 工程质量
- **新增 CI 工作流** - 增加 Rust `cargo check --all-targets --locked`、`cargo test --locked`，以及后台和默认主题前端构建检查。
- **前端锁文件纳入提交** - 放开 `web/pnpm-lock.yaml` 和 `themes/default/pnpm-lock.yaml`，保证 CI 的 `--frozen-lockfile` 可复现。
- **前端低风险清理** - 移除后台未使用的 `@dnd-kit/*` 依赖和默认主题未使用的 `axios`，并收紧少量 `any` 类型，不调整路由和核心交互。

### 文档
- **插件开发文档更新** - 补充通用插件代理禁用、WASM 网络限制、安装包安全约束和插件迁移事务化说明。
